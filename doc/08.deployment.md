# `dotenv`

```bash  
npm install --save-dev dotenv
```  

`webpack.config.ts` 파일에서 설정을 불러올 수 없어서 `dotenv-webpack`은 사용하지 않는다.

`webpack.config.ts` 파일 수정

```ts
// (1) 추가
import dotenv from 'dotenv';


const config: webpack.Configuration = {
    // ... 생략
    plugins: [
        // (2) 이 부분 추가
        // dotenv + Webpack.DefinePlugin = dotenv-webpack
        new webpack.DefinePlugin({
            'process.env': JSON.stringify(dotenv.config().parsed),
            'process.env.NODE_ENV': JSON.stringify(process.env.NODE_ENV),
        }),
        new HtmlWebpackPlugin({
            template: './index.html',
            // (3) 이 부분 추가 .env에서 읽은 설정을 HTML 문서에 주입 가능
            title: process.env.SITE_TITLE,
            meta: {
                viewport: 'width=device-width, initial-scale=1',
                'theme-color': process.env.THEME_COLOR || '#ffffff',
            },
            minify: prod,
        }),
        new MiniCssExtractPlugin(),
    ],

    // ... 생략
};
```

```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <!-- .env에서 읽은 설정값을 HTML 템플릿에 주입 -->
    <title><%= htmlWebpackPlugin.options.title %></title>
</head>
<body>
<noscript>애플리케이션 실행을 위해 반드시 자바스크립트를 허용해야 합니다.</noscript>
<div id='root'>앱 로딩 중 ...</div>
</body>
</html>
```

# cssnano

운영 배포 환경에서 CSS 파일의 크기를 압축한다.

```bash  
npm install --save-dev cssnano
```

`postcss.config.ts` 파일 수정

```ts
module.exports = {
    plugins: {
        tailwindcss: {},
        autoprefixer: {},
        // 아래 줄 추가
        ...(process.env.NODE_ENV === "production" ? { cssnano: {} } : {}),
    },
};
```

# 우분투에서 nvm 설치

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
source ~/.bashrc
```

2024년 9월 22일 현재 최신 버전은 0.40.1 버전이며 이는 변경될 수 있다.

LTS 버전 설치

```bash
nvm install --lts
nvm alias default lts/*
nvm use default
```

설치 버전 목록 확인

```bash
nvm ls
```

현재 사용 중 버전 확인

```bash
nvm current
```

nodejs 및 npm 버전 확인

```bash
node -v
npm -v
```

## Nginx 설정

### 단순 HTML 파일 서비스

```
server {
        listen 80;
        listen [::]:80;
        charset utf-8;

        server_name www.road2gm.co.kr road2gm.co.kr;
        root /var/www/road2gm/repo;

        location / {
                try_files $uri $uri/ =404;
        }

        access_log /var/www/road2gm/logs/access.log;
        error_log /var/www/road2gm/logs/error.log;
}

server {
        listen 443 ssl;
        listen [::]:443;
        charset utf-8;

        server_name www.road2gm.co.kr road2gm.co.kr;
        root /var/www/road2gm/repo;

        ssl_certificate /var/www/road2gm/ssl/road2gm.crt;
        ssl_certificate_key /var/www/road2gm/ssl/road2gm.key;

        location / {
                try_files $uri $uri/ =404;
        }

        access_log /var/www/road2gm/logs/access.log;
        error_log /var/www/road2gm/logs/error.log;
}
```

### 리액트 서비스

```
server {
	listen 80;
	listen [::]:80;
	charset utf-8;

	server_name www.road2gm.co.kr road2gm.co.kr;
	root /var/www/road2gm/repo;

	location /assets {
		root /var/www/road2gm/repo/public;
	   	try_files $uri =404;
		access_log off;
		log_not_found off;
	}

	location / {
		# react-router-dom 하위 도메인 url 접근 시 404 not-found 오류 처리
		root /var/www/road2gm/repo/dist;
	   	index index.html;
	    try_files $uri /index.html;
	   	add_header Cache-Control "no-store, no-cache, must-revalidate";
	   	expires off;
	}

	access_log /var/www/road2gm/logs/access.log;
	error_log /var/www/road2gm/logs/error.log;
}
```
